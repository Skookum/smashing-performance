5 Things You Should Know Before Starting a Mobile HTML5 Project

Here we've collected the demos for Smashing Magazine's mobile web performance article. All demos are meant to be run on a mobile device. To check out the code yourself, run:

$ git clone git://github.com/Skookum/smashing-performance.git
$ cd smashing-performance

DOM Complexity

This example implements a simple tab UI and measures response time to taps against DOM size per device. Try it on your device: open the page, tab back and forth, double the elements, tab back and forth, and repeat. Eventually, no matter how fast your hardware is, navigation speed will suffer. All data is funneled back to SDW via Google Analytics

We used this experiment to determine that successful HTML5 experiences are limited to about 480 active elements.
